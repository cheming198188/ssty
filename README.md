# 神兽体育训练课件与报告平台

一个纯 Python 标准库实现的 Web 应用，面向健身房教练为幼儿、青少年与成人生成训练课件、周期计划、学员档案、单节课教案和家长训练报告。

## 功能

- 学员建档：录入年龄、目标、评估情况、训练需求和限制条件，并可保存为档案列表
- 智能生成：按训练目标、周期和单次课时生成周期计划、真实课次模板、动作要求和示范视频入口
- 课时支持：可选 `60` 分钟或 `90` 分钟，训练内容会按时长完整编排
- 单课打印：支持单节课教案独立打印，适配 A4 纸张
- 训练记录：保存每次课的完成内容、投入度、RPE、教练备注和家庭建议
- 媒体归档：支持上传训练照片和短视频并随报告一起保存
- 家长沟通：自动生成更适合直接发送家长的图文版训练汇报和微信长图版

## 适用场景

- 早教
- 私教课
- 小班课/团课
- 专项（闯关）训练
- 跑步训练

## 训练对象

- 幼儿
- 青少年
- 成人

## 启动

```bash
python3 app.py
```

默认运行在：

```text
http://127.0.0.1:8899
```

也支持通过环境变量覆盖监听地址和端口：

```bash
HOST=0.0.0.0 PORT=8899 ENABLE_RELOAD=0 python3 app.py
```

## 输出目录

- 学员档案保存在 `output/athletes`
- 训练报告保存在 `output/reports`
- 上传的训练照片和短视频保存在 `output/media`
- 打印或导出相关文件可放在 `output/doc`

## 说明

- 默认启用自动重启，修改 `app.py`、`static/app.js`、`static/styles.css` 后会自动刷新服务。
- 线上部署时建议设置 `ENABLE_RELOAD=0`，避免自动重载拉起子进程影响平台运行。
- 当前版本使用内置训练模板规则生成课件，适合作为教练工作台原型和后续功能扩展基础。
- 如果要进一步投入真实业务使用，下一步建议接入账号体系、云存储和可打印 PDF 导出。

## Render 发布

项目已包含 `render.yaml`，可以直接用于 Render Web Service 部署。

推荐流程：

1. 把项目上传到 GitHub 仓库
2. 登录 Render 并选择 `New +` -> `Blueprint`
3. 选择你的 GitHub 仓库
4. Render 会自动识别 `render.yaml` 并创建服务
5. 首次部署完成后，使用 Render 自动分配的公网域名访问

注意事项：

- Render 磁盘默认不是持久化存储，`output/athletes`、`output/reports`、`output/media` 中的数据在服务重建后可能丢失。
- 如果你准备正式使用，建议下一步把档案、报告和媒体迁移到对象存储或数据库。

## 上传到 GitHub 前

项目已经包含 `.gitignore`，会自动忽略：

- 本地缓存和临时目录
- 学员档案 JSON
- 训练报告 JSON
- 上传的图片和视频

这样上传仓库时，不会把真实会员数据和本地运行垃圾文件一起带上去。
